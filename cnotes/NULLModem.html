The null Modem Cable



Quite frequently there is a need to connect two computers (DTEs) that are very close to each other using RS-233-C. Since neither is a modem (DCE), there is an interface problem to consider. This problem can be solved using a device called a null modem. A null modem device is a cable that connects the transmit line of one serial port to the receive line of the other. It also crosses some of the other lines in a similar fashion.

For most applications pins 1 through 8 plus 20 and 22 are all that are needed. The table below shows the basic pins that would be required for a null modem connection.
 
 
 

Pin # Signal Description
2 TX Transmit Data
3 RX Receive Data
4 RTS Request to Send
5 CTS Clear to Send
20 DTR Data Terminal Ready
6 DSR Data Set Ready
8 DCD Data Carrier Detect

 

In constructing the cable bear in mind that pin 2 (Tx) is an output at both ends so it must be crossed over to pin 3 (Rx) at the other end. Similar care must be taken in connecting the control lines. The RTS is an output and its corresponding input will be the CTS line. This leaves the DCD and many devices require that this pin be "on" before data transfer can take place. The DCD can be simulated with a signal from some other pin.
